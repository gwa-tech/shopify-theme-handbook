Design & Content Experimentation for Shopify Themes

Great Shopify themes aren't born — they're refined through experimentation. This chapter teaches you the professional techniques developers use to test, iterate, and perfect theme designs. You'll learn how to create demo content, test layout variations, and make data-informed design decisions.

📌 Chapter Overview

Time estimate: 60 – 90 minutes
Tags: CSS UX Design

Why Design Experimentation Matters

Amateur developers build a feature once and ship it. Professional developers build three variations, test them, and ship the best one. Experimentation is what separates functional themes from premium-quality themes.

Experimentation helps you:

  • Discover better solutions — Your first idea is rarely your best idea
  • Build a design intuition — Pattern recognition comes from seeing many variations
  • Justify decisions — "I chose this layout because I tested three options and this one performed best"
  • Speed up future work — Once you've tested patterns, you know what works
  • Impress clients — Showing multiple options demonstrates professionalism
✅ The Experiment Branch Workflow

Always use Git branches for design experiments. Create a branch like experiment/hero-v2, try your ideas, and compare them against the current version. If the experiment wins, merge it. If not, you haven't broken anything. This is how professional teams work.

Creating Realistic Demo Content

Your theme is only as good as the content it showcases. Testing with "Lorem ipsum" and placeholder squares gives you a false picture. Here's how professionals create demo content:

Product Catalog Strategy

Create a diverse product catalog that tests edge cases:

💡 Create Multiple Demo Stores

Professional theme developers create multiple development stores, each with a different niche: fashion, electronics, food, beauty, furniture. This lets you test how your theme looks across different product types and photography styles. Shopify Partner accounts allow unlimited free dev stores.

Collection Structure for Testing

Create these collections in your development store:

  • All Products — Auto-generated, tests large grids and pagination
  • New Arrivals — 8–12 products, tests standard collection page
  • Best Sellers — 6–8 products, tests featured collection sections
  • Sale — Products with compare-at prices, tests sale displays
  • Single Product Collection — Just 1 product, tests edge cases
  • Empty Collection — No products, tests empty state handling
  • Large Collection — 50+ products, tests pagination and performance

Typography Scale Testing

Professional themes use a consistent type scale. Here's a recommended scale to start with and adjust:

Typography Scale (CSS Custom Properties) 
:root {
  /* Modular scale ratio: 1.25 (Major Third) */
  --font-size-xs: 0.75rem;    /* 12px — captions, badges */
  --font-size-sm: 0.875rem;   /* 14px — small text, meta info */
  --font-size-base: 1rem;     /* 16px — body text */
  --font-size-md: 1.125rem;   /* 18px — lead text */
  --font-size-lg: 1.25rem;    /* 20px — h4, card titles */
  --font-size-xl: 1.5rem;     /* 24px — h3 */
  --font-size-2xl: 1.875rem;  /* 30px — h2 */
  --font-size-3xl: 2.25rem;   /* 36px — h1 on content pages */
  --font-size-4xl: 3rem;      /* 48px — hero headings */
  --font-size-5xl: 3.75rem;   /* 60px — large hero headings */

  /* Line heights */
  --line-height-tight: 1.2;    /* Headings */
  --line-height-snug: 1.4;     /* Sub-headings */
  --line-height-normal: 1.6;   /* Body text */
  --line-height-relaxed: 1.8;  /* Long-form content */

  /* Letter spacing */
  --tracking-tight: -0.025em;  /* Large headings */
  --tracking-normal: 0;        /* Body text */
  --tracking-wide: 0.05em;     /* Uppercase labels */
  --tracking-wider: 0.1em;     /* Nav items, badges */
}
✅ Typography Testing Checklist

When testing typography, check these scenarios:
□ Headings at all 6 levels (h1–h6)
□ Body text in paragraphs (readability at 60–75 characters per line)
□ Product titles at different lengths
□ Price display at different values ($9.99 vs. $1,299.99)
□ Navigation menu with varying text lengths
□ Mobile readability (minimum 16px body text)
□ Contrast ratios (use Chrome DevTools accessibility checker)

💡 Color Contrast Is Non-Negotiable

Every text/background combination must meet WCAG AA contrast ratios: 4.5:1 for body text, 3:1 for large text. Use WebAIM's Contrast Checker to verify. Poor contrast isn't just an accessibility issue — it directly affects readability and conversion rates.

Responsive Breakpoint Testing

Every layout decision must be tested across devices. Here's a professional testing strategy:

Recommended Breakpoints

Breakpoint System 
/* Mobile-first approach */

/* Small mobile: 320px – 479px */
/* Default styles target this range */

/* Large mobile: 480px – 749px */
@media (min-width: 480px) { }

/* Tablet: 750px – 999px */
@media (min-width: 750px) { }

/* Small desktop: 1000px – 1199px */
@media (min-width: 1000px) { }

/* Desktop: 1200px – 1399px */
@media (min-width: 1200px) { }

/* Large desktop: 1400px+ */
@media (min-width: 1400px) { }

/* Dawn's breakpoints (for reference):
   - 750px: mobile → tablet
   - 990px: tablet → desktop
*/

Device Testing Checklist

Responsive Testing Checklist 
TEST EVERY PAGE AT THESE WIDTHS:
═════════════════════════════════

□ 320px  — iPhone SE (smallest common phone)
□ 375px  — iPhone 12/13/14
□ 414px  — iPhone 14 Plus
□ 390px  — iPhone 14 Pro
□ 768px  — iPad portrait
□ 1024px — iPad landscape
□ 1280px — Common laptop
□ 1440px — Standard desktop
□ 1920px — Large monitor

FOR EACH WIDTH, CHECK:
═════════════════════════════════

□ Text is readable (no overflow, no tiny text)
□ Images aren't stretched or cropped badly
□ Buttons are tappable (minimum 44x44px touch target)
□ Navigation is usable
□ Product grid adapts columns correctly
□ No horizontal scrollbar appears
□ Forms are usable (inputs aren't too small)
□ Spacing feels balanced (not too cramped or too sparse)
□ Modal/drawer overlays cover correctly
□ Sticky elements don't overlap content
💡 Chrome DevTools Device Mode

Press F12 → click the device toolbar icon (or Ctrl+Shift+M). Select preset devices or enter custom dimensions. Enable "responsive" mode and drag the viewport width to find where your design breaks. Fix those breakpoints first — they're more important than fitting specific device widths.

Spacing and Whitespace Experiments

Whitespace is the most underappreciated design tool. Premium themes use generous whitespace to create elegance and focus attention.

Spacing Scale

Spacing Scale (CSS Custom Properties) 
:root {
  /* 4px base unit — all spacing is a multiple of 4 */
  --space-1: 0.25rem;   /* 4px  — tight gaps */
  --space-2: 0.5rem;    /* 8px  — inline spacing */
  --space-3: 0.75rem;   /* 12px — compact spacing */
  --space-4: 1rem;      /* 16px — default spacing */
  --space-5: 1.25rem;   /* 20px — comfortable spacing */
  --space-6: 1.5rem;    /* 24px — section inner padding */
  --space-8: 2rem;      /* 32px — component gaps */
  --space-10: 2.5rem;   /* 40px — section padding */
  --space-12: 3rem;     /* 48px — large section gaps */
  --space-16: 4rem;     /* 64px — section separation */
  --space-20: 5rem;     /* 80px — major section breaks */
  --space-24: 6rem;     /* 96px — dramatic spacing */
}

Test these spacing experiments:

  • Section padding: Try 40px, 60px, 80px, and 100px vertical padding between sections
  • Grid gaps: Test 16px, 20px, 24px, and 32px between product cards
  • Container width: Try 1200px, 1320px, and 1440px max content width
  • Card padding: Test 16px, 20px, and 24px internal card padding
  • Text spacing: Test paragraph margin-bottom at 1rem, 1.25rem, and 1.5rem

Complete Experiment Workflow

1

Define the Hypothesis

"I think a 3-column product grid with portrait (3:4) images will look more premium than the current 4-column square grid."

2

Create an Experiment Branch

WSL Terminal 
git checkout -b experiment/portrait-grid
3

Build the Variation

Implement the change. Test with your demo products. View on multiple screen sizes.

4

Compare Side by Side

Switch between main and experiment/portrait-grid branches. Take screenshots of both. Compare them at the same viewport widths.

5

Decide and Merge (or Discard)

If the experiment wins 
git checkout main
git merge experiment/portrait-grid
git branch -d experiment/portrait-grid
If the experiment loses 
git checkout main
git branch -D experiment/portrait-grid

Key Takeaways

  • Create realistic demo content with diverse product types and edge cases
  • Use Burst, Unsplash, and Pexels for free, legal demo images
  • Test multiple homepage layout compositions before settling on a default
  • Experiment with product grid columns, card designs, and image aspect ratios
  • Test at least 3 font pairings before committing to typography
  • Build a functional color system — not just pretty colors
  • Test multiple header layout variations and mobile nav patterns
  • Create hero banner variations for different use cases
  • Test at all common device widths — find where your design breaks
  • Use a consistent spacing scale based on a 4px unit
  • Use Git branches for every experiment — compare, decide, merge or discard
  • Create multiple dev stores for different niches (fashion, tech, food)
